# 7. 数据库设计

## 7.1 数据库目标

SQLite 数据库只存两个表：

```text
1. user：微信小程序用户表
2. device：硬件设备表
```

数据库不保存 CSI 数据流，不保存实时 MQTT 状态，不保存算法运行中的缓存数据。

数据库关系：

```text
一个 user 可以绑定多个 device
一个 device 当前只属于一个 user
```

---

## 7.2 用户表 user

表名：

```text
user
```

字段设计：

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `wx_openid` | String(64) | Primary Key | 微信用户唯一标识 |
| `wx_unionid` | String(64) | Nullable, Unique | 微信 UnionID，预留 |
| `nickname` | String(64) | Nullable | 用户昵称 |
| `avatar_url` | String(512) | Nullable | 用户头像地址 |
| `role` | String(16) | Not Null, Default=`user` | 用户角色 |
| `status` | String(16) | Not Null, Default=`active` | 用户状态 |
| `created_at` | DateTime | Not Null | 创建时间 |
| `updated_at` | DateTime | Not Null | 更新时间 |
| `last_login_at` | DateTime | Nullable | 最近登录时间 |

用户状态建议：

```text
active
disabled
```

用户角色建议：

```text
user
admin
```

模型示例：

```python
from datetime import datetime, timezone

from ..extensions import db


def now_utc():
    return datetime.now(timezone.utc)


class User(db.Model):
    __tablename__ = "user"

    wx_openid = db.Column(db.String(64), primary_key=True)
    wx_unionid = db.Column(db.String(64), unique=True, nullable=True)

    nickname = db.Column(db.String(64), nullable=True)
    avatar_url = db.Column(db.String(512), nullable=True)

    role = db.Column(db.String(16), nullable=False, default="user")
    status = db.Column(db.String(16), nullable=False, default="active")

    created_at = db.Column(db.DateTime(timezone=True), nullable=False, default=now_utc)
    updated_at = db.Column(db.DateTime(timezone=True), nullable=False, default=now_utc, onupdate=now_utc)
    last_login_at = db.Column(db.DateTime(timezone=True), nullable=True)

    devices = db.relationship("Device", back_populates="owner")
```

## 7.3 设备表 device

表名：

```text
device
```

字段设计：

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `device_name` | String(32) | Primary Key | 硬件初始化确定的设备名称 |
| `owner_openid` | String(64) | Foreign Key | 绑定用户，对应 `user.wx_openid` |
| `display_name` | String(64) | Nullable | 管理页面显示名称 |
| `remark` | String(255) | Nullable | 管理备注 |
| `location` | String(64) | Nullable | 安装位置 |
| `status` | String(16) | Not Null, Default=`enabled` | 数据库层面的设备状态 |
| `created_at` | DateTime | Not Null | 创建时间 |
| `updated_at` | DateTime | Not Null | 更新时间 |
| `last_seen_at` | DateTime | Nullable | 最近在线时间，预留 |

设备状态建议：

```text
enabled
disabled
```

设备名规则：

```text
1-32 位，只允许英文字母、数字、下划线 _ 和短横线 -
```

正则：

```text
^[A-Za-z0-9_-]{1,32}$
```

模型示例：

```python
import re
from datetime import datetime, timezone

from ..extensions import db


def now_utc():
    return datetime.now(timezone.utc)


DEVICE_NAME_RE = re.compile(r"^[A-Za-z0-9_-]{1,32}$")


class Device(db.Model):
    __tablename__ = "device"

    device_name = db.Column(db.String(32), primary_key=True)

    owner_openid = db.Column(
        db.String(64),
        db.ForeignKey("user.wx_openid"),
        nullable=False,
        index=True,
    )

    display_name = db.Column(db.String(64), nullable=True)
    remark = db.Column(db.String(255), nullable=True)
    location = db.Column(db.String(64), nullable=True)

    status = db.Column(db.String(16), nullable=False, default="enabled")

    created_at = db.Column(db.DateTime(timezone=True), nullable=False, default=now_utc)
    updated_at = db.Column(db.DateTime(timezone=True), nullable=False, default=now_utc, onupdate=now_utc)
    last_seen_at = db.Column(db.DateTime(timezone=True), nullable=True)

    owner = db.relationship("User", back_populates="devices")

    @staticmethod
    def is_valid_device_name(device_name: str) -> bool:
        return bool(DEVICE_NAME_RE.fullmatch(device_name or ""))
```

## 7.4 数据库边界

数据库只保存：

```text
用户身份信息
设备基础信息
用户与设备绑定关系
```

数据库不保存：

```text
CSI 原始数据
MQTT 实时消息流
算法缓存
WebSocket 临时状态
```

MQTT 服务后续只需要从 `device` 表读取：

```text
device_name
status
```

只处理：

```text
status = enabled
```

的设备。

## 9.4 管理员登录

第一阶段使用 `.env` 中的 `ADMIN_PASSWORD` 作为后台密码。

不要新增 admin 表。

示例路由逻辑：

```python
from flask import Blueprint, current_app, redirect, render_template, request, session, url_for, flash

admin_bp = Blueprint("admin", __name__, url_prefix="/admin")


@admin_bp.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        password = request.form.get("password", "")
        if password == current_app.config["ADMIN_PASSWORD"]:
            session["admin_logged_in"] = True
            return redirect(url_for("admin.index"))
        flash("管理员密码错误")
    return render_template("admin/login.html")


@admin_bp.route("/logout", methods=["POST"])
def logout():
    session.pop("admin_logged_in", None)
    return redirect(url_for("admin.login"))
```

登录校验装饰器：

```python
from functools import wraps

from flask import redirect, session, url_for


def admin_required(view_func):
    @wraps(view_func)
    def wrapper(*args, **kwargs):
        if not session.get("admin_logged_in"):
            return redirect(url_for("admin.login"))
        return view_func(*args, **kwargs)
    return wrapper
```

## 9.8 新增设备

路由：

```text
GET /admin/devices/new
POST /admin/devices/new
```

表单字段：

```text
device_name
owner_openid
display_name
location
remark
status
```

校验规则：

```text
device_name 必填
device_name 必须符合 ^[A-Za-z0-9_-]{1,32}$
device_name 不能重复
owner_openid 必须存在于 user 表
status 只能是 enabled 或 disabled
```

示例：

```python
@admin_bp.route("/devices/new", methods=["GET", "POST"])
@admin_required
def new_device():
    users = User.query.order_by(User.created_at.desc()).all()

    if request.method == "POST":
        device_name = request.form.get("device_name", "").strip()
        owner_openid = request.form.get("owner_openid", "").strip()
        display_name = request.form.get("display_name", "").strip() or None
        location = request.form.get("location", "").strip() or None
        remark = request.form.get("remark", "").strip() or None
        status = request.form.get("status", "enabled")

        if not Device.is_valid_device_name(device_name):
            flash("设备名格式错误：只能包含英文字母、数字、下划线和短横线，长度 1-32 位")
            return render_template("admin/device_form.html", users=users, device=None)

        if Device.query.get(device_name):
            flash("设备名已存在")
            return render_template("admin/device_form.html", users=users, device=None)

        owner = User.query.get(owner_openid)
        if not owner:
            flash("绑定用户不存在")
            return render_template("admin/device_form.html", users=users, device=None)

        if status not in {"enabled", "disabled"}:
            flash("设备状态非法")
            return render_template("admin/device_form.html", users=users, device=None)

        device = Device(
            device_name=device_name,
            owner_openid=owner_openid,
            display_name=display_name,
            location=location,
            remark=remark,
            status=status,
        )
        db.session.add(device)
        db.session.commit()
        flash("设备创建成功")
        return redirect(url_for("admin.devices"))

    return render_template("admin/device_form.html", users=users, device=None)
```

## 9.9 编辑设备

路由：

```text
GET /admin/devices/<device_name>/edit
POST /admin/devices/<device_name>/edit
```

可编辑字段：

```text
owner_openid
display_name
location
remark
status
```

不建议编辑：

```text
device_name
```

原因：

```text
device_name 是硬件初始化时确定的名称，也是 MQTT Topic 和 Client ID 生成基础。
如果修改 device_name，容易导致数据库、硬件和 MQTT Topic 不一致。
```

如果设备名写错，建议禁用旧设备记录，再新增正确设备。

## 9.10 启用 / 禁用

禁用设备只修改数据库状态：

```text
device.status = disabled
```

不删除数据库记录。

示例：

```python
@admin_bp.route("/devices/<device_name>/disable", methods=["POST"])
@admin_required
def disable_device(device_name):
    device = Device.query.get_or_404(device_name)
    device.status = "disabled"
    db.session.commit()
    flash("设备已禁用")
    return redirect(url_for("admin.devices"))
```

启用类似：

```python
@admin_bp.route("/devices/<device_name>/enable", methods=["POST"])
@admin_required
def enable_device(device_name):
    device = Device.query.get_or_404(device_name)
    device.status = "enabled"
    db.session.commit()
    flash("设备已启用")
    return redirect(url_for("admin.devices"))
```

用户启用 / 禁用同理：

```text
user.status = active
user.status = disabled
```

## 10.5 原生 JS 优化

只需要少量 JS。

`admin.js` 建议功能：

```text
1. 表单提交前确认。
2. 禁用 / 启用操作确认。
3. 自动隐藏 flash 提示。
```

示例：

```javascript
document.addEventListener("DOMContentLoaded", () => {
  document.querySelectorAll("form[data-confirm]").forEach((form) => {
    form.addEventListener("submit", (event) => {
      const message = form.getAttribute("data-confirm") || "确定执行该操作吗？";
      if (!window.confirm(message)) {
        event.preventDefault();
      }
    });
  });

  document.querySelectorAll(".flash-item").forEach((item) => {
    window.setTimeout(() => {
      item.style.display = "none";
    }, 3000);
  });
});
```

不要加入复杂前端状态管理。

# 11. MQTT 与数据库的衔接规则

虽然本次不实现 MQTT 服务，但数据库设计必须为后续 MQTT 服务留好接口。

后续 MQTT 服务从 `device` 表读取启用设备：

```python
enabled_devices = Device.query.filter_by(status="enabled").all()
```

每个设备生成：

```text
Python Client ID:
python_{device_name}_001

ESP Client ID:
esp_{device_name}_001
```

Topic 规则：

```text
csi/v1/devices/{device_name}/up/online
csi/v1/devices/{device_name}/up/wifi
csi/v1/devices/{device_name}/up/status
csi/v1/devices/{device_name}/up/csi
csi/v1/devices/{device_name}/up/ack
csi/v1/devices/{device_name}/up/fault
csi/v1/devices/{device_name}/down/control
```

数据库只保存 `device_name`，不要保存完整 Topic，也不要保存完整 Client ID。
